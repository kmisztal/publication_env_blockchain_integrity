\section{Conclusions}

This paper presented a lightweight proof-of-concept for evaluating blockchain-inspired data integrity mechanisms in distributed environmental monitoring systems. The study compared conventional storage, audit trails, audit trails with hash chains, and audit trails with hash chains plus provenance and permission reconstruction.

Using a bounded OpenAQ dataset and controlled tampering scenarios, the evaluation produced a threat-coverage matrix that distinguishes model capabilities from expected limitations. The results show that audit trails, hash-chain continuity, and provenance/permission reconstruction address different classes of integrity threats. Conventional storage provides limited post-hoc integrity evidence; audit trails improve local event verification; hash chains improve stream-continuity verification; and provenance/permission reconstruction supports actor-related and governance-related checks.

The contribution is intentionally limited to information-systems integrity evaluation. The work does not claim environmental outcomes, air-quality findings, production blockchain readiness, or regulatory compliance. Its value lies in providing a reproducible structure for comparing integrity models and verification workflows using realistic environmental monitoring data as the application substrate.
